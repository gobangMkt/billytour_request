# 릴스 단건 상세페이지 — 설계 문서

작성일: 2026-06-15
상태: 설계 확정 대기

---

## 1. 배경 / 목표

빌리투어 글로벌(40만) 중심 상품이 전혀 팔리지 않음. 노선을 **릴스(숏폼) 단건 판매 집중**으로
전환한다. 기존 글로벌 랜딩은 **그대로 유지**하고, 릴스 전용 상세페이지를 **신규 제작 + 별도 배포**한다.

**핵심 메시지(FOMO):** "원장님 방, 아직도 한국어 영상/블로그만으로 끝인가요? 요즘 세입자는
릴스·숏츠로 방을 찾습니다. 안 하면 노출에서 밀립니다." → 기존 빌리투어 영상 재활용으로
릴스 1개를, 초기기념 50% 할인 **5만원**에.

## 2. 확정 스펙

| 항목 | 내용 |
|------|------|
| 기존 글로벌 랜딩 | **유지**(건드리지 않음), 기존 배포 그대로 |
| 새 페이지 | 릴스 단건 전용 상세페이지 신규 + **별도 Vercel 배포(독립 URL)** |
| 상품 | 릴스(숏폼) 1개 제작 → 유튜브 Shorts·커뮤니티 + 인스타 Reels 동시 업로드 |
| 제작 방식 | **기존 빌리투어 영상 재활용 (추가 촬영 없음)** |
| 가격 | **50,000원 (VAT 별도)** · 초기기념 50% 할인 · 정가 100,000원 |
| 타겟 | **기존 원장님 전용** (이미 빌리투어 찍은 분 → 재활용할 영상 보유) |
| 톤 | 페인포인트·FOMO 자극형. "릴스 안 하면 손해", "아직 블로그만?" |

## 3. 구현 방식 (A안)

`landing/` 폴더 안에 릴스 entry를 추가한다. 기존 글로벌 자산은 손대지 않는다.

**신규 파일**
- `landing/reels.html` — 릴스 페이지 진입점 (React CDN + Babel Standalone, 기존 index.html 구조 복제)
- `landing/variant-reels.jsx` — 릴스 페이지 본문 컴포넌트

**재활용 (수정 없음)**
- `landing/shared.jsx` — LandingNav, FaqItem, REQUEST_URL, CHANNEL_URL 등 공용
- `landing/shared.css`, `landing/colors_and_type.css` — 디자인 토큰/공용 스타일

> 글로벌 전용 컴포넌트(CompareTable, GlobalPanel 등)는 reels 페이지에서 import/사용하지 않음.
> 릴스 전용 섹션 컴포넌트는 variant-reels.jsx 내부에 둔다.

**배포**
- 별도 Vercel 프로젝트로 `reels.html`을 루트로 배포 → 독립 URL.
- 기존 글로벌 Vercel 배포는 그대로 유지.

## 4. 페이지 섹션 흐름

1. **HERO** — "원장님 방, 아직도 한국어 영상 하나로 끝인가요?"
   세로 릴스 비주얼 + `정가 10만 ❌ → 5만원` 할인 배지 + 메인 CTA.
2. **공감·자극 (페인포인트)** — "블로그·플레이스만으로 충분하다고요?"
   경쟁 원장 vs 나 대비. 안 하면 노출/문의에서 밀린다.
3. **왜 릴스인가 (트렌드)** — 숏폼 소비 급증, 알고리즘이 릴스/Shorts를 밀어줌,
   체류·도달 우위. (정량 수치는 과장 없이 일반 트렌드 사실 범위로.)
4. **이게 다 됩니다 (상품 정의)** — 기존 영상 → 릴스 1개 제작 → 유튜브 Shorts·커뮤니티
   + 인스타 Reels 동시 업로드. **추가 촬영 없음** 강조.
5. **가격 오퍼** — 정가 100,000원 ❌ → **50,000원** (초기기념 50%, VAT 별도). 한시성 강조.
6. **프로세스** — 신청 → 담당자 확인 → 결제 → 영상 확인 → 릴스 제작 → 업로드 완료.
   (촬영 단계 없어 글로벌보다 짧음.)
7. **FAQ** — "기존 영상 없으면?" / "어떤 영상이든 되나?" / "추가 비용 있나?" /
   "어디에 올라가나?" / "기존 원장님 기준?" 등.
8. **Footer CTA + 채널톡 배너 + 상시 플로팅 CTA** — 기존 패턴 재활용.

디자인 토큰은 U 시그니처 블루 팔레트(`colors_and_type.css`) 그대로 상속.
정형화된 AI 스타일(무분별 라운딩·보라 그라데이션) 금지.

## 5. 신청폼 / 백엔드 처리

- **신청폼:** 기존 `index.html` 신청폼을 **재활용**하되, 글로벌·숏츠 상품 분기를 떼어내고
  **릴스 단건으로 고정**한다(상품 = `릴스단건` 프리셋). 릴스 페이지 CTA → 기존 폼 진입 시
  상품이 릴스로 고정된 상태.
- **GAS:** 기존 `Code.gs`에 릴스 상품 분기 추가. 상품 컬럼(G)에 `릴스단건` 값 처리 +
  결제링크 매칭 로직에 릴스용 분기 추가.

### ⚠️ 결제링크 임시 처리 (반드시 교체 필요)
- 현재 릴스 5만원(VAT포함 55,000원)용 토스링크가 **없음**.
- 임시로 **기존 숏츠단건 링크(110,000원)** 를 연결한다. → **페이지 표시가(5만) ≠ 실결제가(11만) 불일치.**
- **실제 알림톡 발송·결제 오픈 전 반드시 55,000원 새 토스링크 생성 후 교체.**
  - 교체 위치: GAS `Code.gs`의 릴스 결제링크 상수 + 페이지 CTA(있으면).

## 6. 범위 밖 (YAGNI)

- 글로벌 랜딩/상품 수정 — 안 함.
- 신규 원장(영상 미보유) 대상 촬영 플로우 — 안 함(기존 원장 전용).
- 신청폼 신규 제작 — 안 함(기존 재활용).
- 새 GAS 스프레드시트/엔드포인트 — 안 함(기존 확장).

---

## 7. 오픈 이슈 / 확인 필요

1. 결제링크 교체 시점 — 사장님 토스 새 링크 생성 일정.
2. 트렌드 수치(섹션 3) — 인용 가능한 출처 쓸지, 일반 서술로 둘지.
3. 신규 배포 URL/Vercel 프로젝트명 — 배포 단계에서 확정.
